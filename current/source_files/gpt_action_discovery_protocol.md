Use this file only for discovering and ranking behavior-changing actions. Main Project Instructions control behavior.

Trigger Action Discovery when the user asks for better actions, mechanisms, combinations, system improvements, project redesign, audit-plus-patch, rule restoration, anti-regression, maximum-quality changes, or why prior fixes failed.

Define goal and constraints: target system; user; use cases; authority layer; safety; source needs; tools; output obligations; file limit; instruction limit; inheritance; active package state.

Inspect visible material before ranking: main instruction, active files, old versions only as evidence, missing material, current official sources when material.

Generate candidate actions across: main-instruction edit; existing-file edit; new file; split; merge; delete; rename; compression; template; test; regression smoke test; source/knowledge file; delivery rule; safety gate; tool/action workflow; checklist/SOP; no-GPT/human workflow; child-propagation mechanism; Builder/Auditor split.

Combination rule: test combinations, not only single edits. Strong packages may require short main trigger, owner governance file, executable test, delivery template, registry entry, and child propagation rule.

Reject candidates that are unsafe, unverifiable, decorative, non-operational, redundant without layer purpose, high-drift without synchronization, untestable, source-unsupported, too broad, lower-authority-only for critical behavior, or harmful to child systems.

Rank by expected quality gain, reliability, safety, source honesty, testability, maintainability, reversibility, drift reduction, cost/load, and child propagation. Do not rank less work/fewer files above quality or stability.

File-structure action rule: compare edit/new/split/merge/delete/no-file/SOP. New files are preferred when they create distinct owner-rule, separate source freshness, isolate tests, reduce conflict, improve inheritance, or prevent mixed-responsibility dump.

Deletion action rule: deleting a rule/file is a candidate action with burden of proof. It must pass Patch Lock, deletion burden, and protected-registry tests before delivery.

Anti-regression action rule: compression/rewrite must run Patch Lock and protected-registry regression before final delivery; if behavior is weakened, restore, replace stronger, or state Not Ready.

Major Action Discovery output: goal; constraints; inspected material; action map; selected mechanism/combination; rejected alternatives; patch summary; executed tests; risks; verdict.
